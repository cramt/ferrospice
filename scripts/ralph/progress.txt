## Codebase Patterns
- AC analysis: `simulate_ac(&Netlist) -> Result<SimResult>` in ferrospice-core::ac
- AC uses `ComplexLinearSystem` (separate real/imag sparse matrices) solved via faer `c64` LU decomposition
- AC sweep modes: DEC (n pts/decade, log spacing), OCT (n pts/octave), LIN (n total points, linear)
- AC source excitation: `AcSpec { mag, phase }` on voltage/current sources â†’ complex RHS in branch/node equations
- AC capacitor stamp: imaginary part += Ï‰C (susceptance) between cap nodes
- AC inductor stamp: imaginary part += -Ï‰L on branch diagonal (branch equation)
- AC diode stamp: small-signal conductance g_d at DC OP + junction capacitance Cj(V) + transit time cap Ï„*g_d
- `DiodeModel::junction_capacitance(v)` computes depletion cap Cj(V) = CJO/(1-V/VJ)^M with linear extrapolation beyond FC*VJ
- Adaptive timestep: LTE estimated from BE vs Trap difference on capacitor charges and inductor fluxes
- `BreakpointTable` collects discontinuity times from waveforms, forces timestep to land on them
- `waveform::breakpoints(&Waveform, &TranParams) -> Vec<f64>` extracts breakpoint times from waveforms
- Breakpoint behavior: at breakpoints, use BE integration and reduce step by 0.1Ã— for stability
- `h_max` = tmax from .tran if specified, otherwise min(tstep, tstop/50) â€” caps internal timestep
- LTE tolerance: `TRTOL * max(vol_tol, chg_tol)` where TRTOL=7.0 (matching ngspice CKTtrtol)
- Timestep adjustment: reject if new_h < 0.9Ã—step_h, accept and grow up to 2Ã— otherwise
- step_h vs h: step_h is the actual step size used this iteration, h is the suggested next step from LTE
- Waveform evaluation: `waveform::evaluate(&Waveform, t, &TranParams) -> f64` in ferrospice-core::waveform
- `TranParams { tstep, tstop }` provides defaults for waveform parameters (rise/fall times, SIN freq, etc.)
- `VoltageSourceInstance` and `CurrentSourceInstance` in MnaSystem track branch_idx/node indices + waveform
- In transient, after copying base stamps, waveform values override source RHS entries at each timestep
- PULSE defaults: td=0, tr=tstep, tf=tstep, pw=0, per=tr+pw+tf; repeats via time modulo period
- SIN phase parameter is in degrees (converted to radians internally), damping via exp(-theta*(t-td))
- Transient: `simulate_tran(&Netlist) -> Result<SimResult>` in ferrospice-core::transient
- Transient uses BE for first timestep, then switches to Trapezoidal (matching ngspice)
- Capacitor companion model: BE â†’ geq=C/h, ieq=-C/h*v_prev; Trap â†’ geq=2C/h, ieq=-(2C/h*v_prev + i_prev)
- Inductor companion model: BE â†’ req=L/h, veq=-L/h*i_prev; Trap â†’ req=2L/h, veq=-(2L/h*i_prev + v_prev)
- Inductor in transient: adds -req on branch diagonal, veq on branch RHS (on top of topology stamps)
- `CapacitorInstance` and `InductorInstance` in MnaSystem hold resolved values and IC params
- IC= param on C/L elements overrides DC OP initial voltage/current for transient
- Transient output: "time" vector + node voltages + branch currents, plot name "tran1"
- DC sweep: `simulate_dc(&Netlist) -> Result<SimResult>` in ferrospice-core::simulate
- DC sweep reuses assembled MNA matrix, only modifies RHS entries per sweep point
- Voltage source sweep: set `system.rhs[num_nodes + vsource_idx] = sweep_value`
- Current source sweep: undo old stamp, apply new value at node RHS entries
- DC sweep result has sweep variable as first vector (e.g., "v1"), then node voltages and branch currents
- Double sweep: outer loop is src2, inner loop is src1
- `NodeMap::len()` and `NodeMap::is_empty()` are now public
- Integration tests go in `ferrospice-core/tests/` â€” use `CARGO_MANIFEST_DIR` to locate ngspice test files
- ngspice check.sh filters: SPARSE|KLU|CPU|Dynamic|Note|Circuit|Doing|---|Data|Index|trans|acan|etc.
