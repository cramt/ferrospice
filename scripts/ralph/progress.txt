## Codebase Patterns
- Transient: `simulate_tran(&Netlist) -> Result<SimResult>` in ferrospice-core::transient
- Transient uses BE for first timestep, then switches to Trapezoidal (matching ngspice)
- Capacitor companion model: BE → geq=C/h, ieq=-C/h*v_prev; Trap → geq=2C/h, ieq=-(2C/h*v_prev + i_prev)
- Inductor companion model: BE → req=L/h, veq=-L/h*i_prev; Trap → req=2L/h, veq=-(2L/h*i_prev + v_prev)
- Inductor in transient: adds -req on branch diagonal, veq on branch RHS (on top of topology stamps)
- `CapacitorInstance` and `InductorInstance` in MnaSystem hold resolved values and IC params
- IC= param on C/L elements overrides DC OP initial voltage/current for transient
- Transient output: "time" vector + node voltages + branch currents, plot name "tran1"
- DC sweep: `simulate_dc(&Netlist) -> Result<SimResult>` in ferrospice-core::simulate
- DC sweep reuses assembled MNA matrix, only modifies RHS entries per sweep point
- Voltage source sweep: set `system.rhs[num_nodes + vsource_idx] = sweep_value`
- Current source sweep: undo old stamp, apply new value at node RHS entries
- DC sweep result has sweep variable as first vector (e.g., "v1"), then node voltages and branch currents
- Double sweep: outer loop is src2, inner loop is src1
- `NodeMap::len()` and `NodeMap::is_empty()` are now public
- Integration tests go in `ferrospice-core/tests/` — use `CARGO_MANIFEST_DIR` to locate ngspice test files
- ngspice check.sh filters: SPARSE|KLU|CPU|Dynamic|Note|Circuit|Doing|---|Data|Index|trans|acan|etc.
- ngspice .out data rows are tab-separated, start with integer index, use scientific notation
- `simulate_op` works on any netlist regardless of what analysis commands are present (ignores .tran/.ac)
- Use `faer` 0.24 for sparse matrix / linear algebra (pure Rust, supports LU via `FullPivLu`)
- Import `faer::prelude::Solve` trait to use `.solve()` on LU decomposition
- `SparseMatrix` uses triplet (COO) form with duplicate summing — standard for MNA stamp accumulation
- `LinearSystem` wraps `SparseMatrix` + RHS vector and provides `.solve()` -> `Result<Vec<f64>>`
- ferrospice-netlist already parses: R, C, L, V, I, D, Q, M, J, K, E, F, G, H, B, X elements
- ferrospice-netlist already parses: .op, .dc, .tran, .ac, .noise, .tf, .sens analyses
- Workspace deps: `thiserror` (workspace), `facet` (workspace), `approx` (dev)
- MNA assembly: `assemble_mna(&Netlist) -> Result<MnaSystem>` in ferrospice-core::mna
- DC OP solver: `simulate_op(&Netlist) -> Result<SimResult, MnaError>` in ferrospice-core::simulate
- SimResult vectors: node voltages as `v(node_name)`, branch currents as `vsource_name#branch` (lowercase)
- Node "0" is ground and excluded from the MNA matrix (NodeMap returns None for it)
- MNA matrix size = num_non_ground_nodes + num_voltage_sources
- Solution vector layout: [node voltages..., vsource branch currents...]
- SPICE current source convention: `I n+ n-` → current exits n+ externally, enters n-
- Use `stamp_conductance()` helper for resistor-like stamps between two nodes
- Inductor in DC is modeled as a 0V voltage source (branch equation), added to `vsource_names`
- Capacitor in DC has no MNA contribution — just register nodes, skip stamping
- `simulate_op` automatically includes inductor branch currents since they share `vsource_names`
- NR solver: `newton_raphson_solve(options, dim, num_nodes, load_system, initial_guess)` in ferrospice-core::newton
- `load_system` callback signature: `Fn(&[f64], &mut LinearSystem, f64)` — (solution, system, source_factor)
- NR convergence: voltages use `reltol * max(|old|,|new|) + vntol`, currents use `abstol` instead of `vntol`
- NR fallback chain: direct NR → Gmin stepping (1e-2 down to 1e-12) → source stepping (0.0 to 1.0)
- Diode companion model: g_d = IS/Vt * exp(V/Vt), i_eq = I(V) - g_d * V; stamp g_d as conductance, i_eq as current source
- `NrOptions::default()` matches ngspice: ABSTOL=1e-12, RELTOL=1e-3, VNTOL=1e-6, ITL1=100, ITL2=50, GMIN=1e-12
- Diode model: `DiodeModel` in ferrospice-core::diode — IS, N, RS, BV, IBV, CJO, VJ, M, TT params
- Diode companion: `(g_eq, i_eq) = model.companion(v_jct)` — stamp g_eq as conductance, i_eq as current source
- `DiodeModel::from_model_def(&ModelDef)` parses .model params; `.with_instance_params(&[Param])` applies overrides
- `DiodeInstance` in MNA: tracks anode_idx, cathode_idx, internal_idx (for RS>0), and resolved DiodeModel
- `stamp_conductance` is now public (used by both mna.rs and simulate.rs NR load function)
- `Triplet` struct and `SparseMatrix::triplets()` are public — NR load copies base linear stamps each iteration
- `SparseMatrixError::SingularMatrix(String)` variant wraps NrError messages
- `simulate_op` auto-detects diodes: linear circuits use direct solve, nonlinear use NR
- `solve_nonlinear_op(mna)` builds NR load closure that replays base stamps + diode companions
- Voltage limiting: `pnjlim(v_new, v_old, vt, vcrit)` prevents NR divergence from exp() overflow
- `collect_solution_into` and `simulate_dc` also detect diodes and use NR for each sweep point
