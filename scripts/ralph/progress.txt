## Codebase Patterns
- AC analysis: `simulate_ac(&Netlist) -> Result<SimResult>` in ferrospice-core::ac
- AC uses `ComplexLinearSystem` (separate real/imag sparse matrices) solved via faer `c64` LU decomposition
- AC sweep modes: DEC (n pts/decade, log spacing), OCT (n pts/octave), LIN (n total points, linear)
- AC source excitation: `AcSpec { mag, phase }` on voltage/current sources → complex RHS in branch/node equations
- AC capacitor stamp: imaginary part += ωC (susceptance) between cap nodes
- AC inductor stamp: imaginary part += -ωL on branch diagonal (branch equation)
- AC diode stamp: small-signal conductance g_d at DC OP + junction capacitance Cj(V) + transit time cap τ*g_d
- `DiodeModel::junction_capacitance(v)` computes depletion cap Cj(V) = CJO/(1-V/VJ)^M with linear extrapolation beyond FC*VJ
- Adaptive timestep: LTE estimated from BE vs Trap difference on capacitor charges and inductor fluxes
- `BreakpointTable` collects discontinuity times from waveforms, forces timestep to land on them
- `waveform::breakpoints(&Waveform, &TranParams) -> Vec<f64>` extracts breakpoint times from waveforms
- Breakpoint behavior: at breakpoints, use BE integration and reduce step by 0.1× for stability
- `h_max` = tmax from .tran if specified, otherwise min(tstep, tstop/50) — caps internal timestep
- LTE tolerance: `TRTOL * max(vol_tol, chg_tol)` where TRTOL=7.0 (matching ngspice CKTtrtol)
- Timestep adjustment: reject if new_h < 0.9×step_h, accept and grow up to 2× otherwise
- step_h vs h: step_h is the actual step size used this iteration, h is the suggested next step from LTE
- Waveform evaluation: `waveform::evaluate(&Waveform, t, &TranParams) -> f64` in ferrospice-core::waveform
- `TranParams { tstep, tstop }` provides defaults for waveform parameters (rise/fall times, SIN freq, etc.)
- `VoltageSourceInstance` and `CurrentSourceInstance` in MnaSystem track branch_idx/node indices + waveform
- In transient, after copying base stamps, waveform values override source RHS entries at each timestep
- PULSE defaults: td=0, tr=tstep, tf=tstep, pw=0, per=tr+pw+tf; repeats via time modulo period
- SIN phase parameter is in degrees (converted to radians internally), damping via exp(-theta*(t-td))
- Transient: `simulate_tran(&Netlist) -> Result<SimResult>` in ferrospice-core::transient
- Transient uses BE for first timestep, then switches to Trapezoidal (matching ngspice)
- Capacitor companion model: BE → geq=C/h, ieq=-C/h*v_prev; Trap → geq=2C/h, ieq=-(2C/h*v_prev + i_prev)
- Inductor companion model: BE → req=L/h, veq=-L/h*i_prev; Trap → req=2L/h, veq=-(2L/h*i_prev + v_prev)
- Inductor in transient: adds -req on branch diagonal, veq on branch RHS (on top of topology stamps)
- `CapacitorInstance` and `InductorInstance` in MnaSystem hold resolved values and IC params
- IC= param on C/L elements overrides DC OP initial voltage/current for transient
- Transient output: "time" vector + node voltages + branch currents, plot name "tran1"
- DC sweep: `simulate_dc(&Netlist) -> Result<SimResult>` in ferrospice-core::simulate
- DC sweep reuses assembled MNA matrix, only modifies RHS entries per sweep point
- Voltage source sweep: set `system.rhs[num_nodes + vsource_idx] = sweep_value`
- Current source sweep: undo old stamp, apply new value at node RHS entries
- DC sweep result has sweep variable as first vector (e.g., "v1"), then node voltages and branch currents
- Double sweep: outer loop is src2, inner loop is src1
- `NodeMap::len()` and `NodeMap::is_empty()` are now public
- Integration tests go in `ferrospice-core/tests/` — use `CARGO_MANIFEST_DIR` to locate ngspice test files
- ngspice check.sh filters: SPARSE|KLU|CPU|Dynamic|Note|Circuit|Doing|---|Data|Index|trans|acan|etc.
- ngspice .out data rows are tab-separated, start with integer index, use scientific notation
- `simulate_op` works on any netlist regardless of what analysis commands are present (ignores .tran/.ac)
- Use `faer` 0.24 for sparse matrix / linear algebra (pure Rust, supports LU via `FullPivLu`)
- Import `faer::prelude::Solve` trait to use `.solve()` on LU decomposition
- `SparseMatrix` uses triplet (COO) form with duplicate summing — standard for MNA stamp accumulation
- `LinearSystem` wraps `SparseMatrix` + RHS vector and provides `.solve()` -> `Result<Vec<f64>>`
- ferrospice-netlist already parses: R, C, L, V, I, D, Q, M, J, K, E, F, G, H, B, X elements
- ferrospice-netlist already parses: .op, .dc, .tran, .ac, .noise, .tf, .sens analyses
- Workspace deps: `thiserror` (workspace), `facet` (workspace), `approx` (dev)
- MNA assembly: `assemble_mna(&Netlist) -> Result<MnaSystem>` in ferrospice-core::mna
- DC OP solver: `simulate_op(&Netlist) -> Result<SimResult, MnaError>` in ferrospice-core::simulate
- SimResult vectors: node voltages as `v(node_name)`, branch currents as `vsource_name#branch` (lowercase)
- Node "0" is ground and excluded from the MNA matrix (NodeMap returns None for it)
- MNA matrix size = num_non_ground_nodes + num_voltage_sources
- Solution vector layout: [node voltages..., vsource branch currents...]
- SPICE current source convention: `I n+ n-` → current exits n+ externally, enters n-
- Use `stamp_conductance()` helper for resistor-like stamps between two nodes
- Inductor in DC is modeled as a 0V voltage source (branch equation), added to `vsource_names`
- Capacitor in DC has no MNA contribution — just register nodes, skip stamping
- `simulate_op` automatically includes inductor branch currents since they share `vsource_names`
- NR solver: `newton_raphson_solve(options, dim, num_nodes, load_system, initial_guess)` in ferrospice-core::newton
- `load_system` callback signature: `Fn(&[f64], &mut LinearSystem, f64)` — (solution, system, source_factor)
- NR convergence: voltages use `reltol * max(|old|,|new|) + vntol`, currents use `abstol` instead of `vntol`
- NR fallback chain: direct NR → Gmin stepping (1e-2 down to 1e-12) → source stepping (0.0 to 1.0)
- Diode companion model: g_d = IS/Vt * exp(V/Vt), i_eq = I(V) - g_d * V; stamp g_d as conductance, i_eq as current source
- `NrOptions::default()` matches ngspice: ABSTOL=1e-12, RELTOL=1e-3, VNTOL=1e-6, ITL1=100, ITL2=50, GMIN=1e-12
- Diode model: `DiodeModel` in ferrospice-core::diode — IS, N, RS, BV, IBV, CJO, VJ, M, TT params
- Diode companion: `(g_eq, i_eq) = model.companion(v_jct)` — stamp g_eq as conductance, i_eq as current source
- `DiodeModel::from_model_def(&ModelDef)` parses .model params; `.with_instance_params(&[Param])` applies overrides
- `DiodeInstance` in MNA: tracks anode_idx, cathode_idx, internal_idx (for RS>0), and resolved DiodeModel
- `stamp_conductance` is now public (used by both mna.rs and simulate.rs NR load function)
- `Triplet` struct and `SparseMatrix::triplets()` are public — NR load copies base linear stamps each iteration
- `SparseMatrixError::SingularMatrix(String)` variant wraps NrError messages
- `simulate_op` auto-detects diodes: linear circuits use direct solve, nonlinear use NR
- `solve_nonlinear_op(mna)` builds NR load closure that replays base stamps + diode companions
- Voltage limiting: `pnjlim(v_new, v_old, vt, vcrit)` prevents NR divergence from exp() overflow
- `collect_solution_into` and `simulate_dc` also detect diodes and use NR for each sweep point
- MOSFET Level 1: `MosfetModel` + `MosfetInstance` + `MosfetCompanion` + `stamp_mosfet()` in ferrospice-core::mosfet
- MOSFET follows BJT pattern: companion() computes linearized model, stamp_mosfet() applies to MNA
- MOSFET source-drain symmetry: mode=+1 (normal) or -1 (reversed when VDS<0)
- MOSFET body effect: Vth = VTO + GAMMA*(sqrt(PHI-VBS)-sqrt(PHI)), gmbs = gm * 0.5*GAMMA/sqrt(PHI-VBS)
- MOSFET mos_limit: VGS limited ±0.5V around VTO, VDS uses quadratic limiting for large changes
- MOSFET AC: gds/gm/gmbs real conductances + gate overlap caps (CGSO*W, CGDO*W, CGBO*L) + junction caps (CBD, CBS)
- MOSFET internal nodes: RD>0 creates internal drain, RS>0 creates internal source (like BJT series resistances)
- `has_nonlinear` checks: `!mna.diodes.is_empty() || !mna.bjts.is_empty() || !mna.mosfets.is_empty()`
- Subcircuit expansion: `flatten_netlist(&Netlist) -> Result<Netlist, SubcktError>` in ferrospice-core::subckt
- Controlled sources: VCVS (E), VCCS (G), CCCS (F), CCVS (H) all stamped as linear elements in base MNA matrix
- VCVS/CCVS add branch equations (like voltage sources); VCCS/CCCS stamp directly into admittance matrix
- F and H elements reference controlling voltage source by name → vsource_offset_map built during first pass
- SPICE convention: `G out+ out- ctrl+ ctrl- gm` drives current FROM out+ TO out- internally (current LEAVES out+ externally)
- Controlled sources are linear → no NR iteration needed, AC/transient get them free from base matrix copy
- Subcircuit flattening is called automatically at the start of `assemble_mna()` — transparent to callers
- Expanded element names: `instance_prefix.element_name` (e.g., "x1.r1", "x1.x2.r1" for nested)
- Expanded internal nodes: `instance_prefix.node_name` (e.g., "x1.mid"); ground "0" never remapped
- `.global` nodes pass through subcircuit expansion unchanged (never prefixed)
- Model defs inside subcircuits are hoisted to top level without name changes
- Noise analysis: `simulate_noise(&Netlist) -> Result<SimResult>` in ferrospice-core::noise
- Noise uses adjoint (transposed) AC MNA solve: inject unit current at output, measure response at noise sources
- Noise contribution formula: noise_density * |V_adj(n1) - V_adj(n2)|² (NevalSrc convention)
- `ComplexLinearSystem::solve_transpose()` solves adjoint (transposed) complex system
- `ResistorInstance` in MnaSystem tracks name/pos_idx/neg_idx/resistance for noise
- Device noise params: KF (flicker coeff), AF (flicker exponent) on DiodeModel, BjtModel, MosfetModel, JfetModel
- Noise result: two plots — noise1 (spectrum: frequency + onoise/inoise_spectrum) + noise2 (integrated: onoise/inoise_total)
- `generate_ac_sweep()` and `stamp_imag_conductance()` are public in ac.rs — reusable by noise
- PZ analysis: `simulate_pz(&Netlist) -> Result<SimResult>` in ferrospice-core::pz
- PZ uses eigenvalue decomposition: poles from det(G+sC)=0, zeros from cofactor determinant
- PZ eigenvalues: G^{-1}*C eigenvalues → poles = -1/λ for non-zero λ; fallback to GEV when G singular
- PZ conjugate dedup: only remove conjugate pairs (opposite imaginary sign), preserve repeated real roots
- PZ relative threshold: use `max_mag * 1e-10` (not absolute 1e-20) to distinguish real eigenvalues from numerical noise
- `ResistorInstance` has `ac_resistance: Option<f64>` — PZ adjusts G matrix with delta conductance
- PZ result: pz1 plot (poles or zeros), pz2 plot (zeros when both computed)
- `build_jacobian()` from tf.rs reused by pz.rs for G matrix construction
