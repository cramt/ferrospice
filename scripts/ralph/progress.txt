## Codebase Patterns
- PZ output: `format_pz_table()` in output.rs merges all pz* plots, formats complex values as `re,\tim` pairs, 2 cols/page
- Sensitivity output: always routes through `format_print_all_table()` (4 cols/page pagination), even when `.print SENS` exists
- TF plots: named sequentially tf1, tf2, ... — `format_tf_output()` takes a single SimPlot directly
- DC sweep vector naming: "v-sweep" for voltage sources, "i-sweep" for current sources (matching ngspice). `find_sweep_vector()` in output.rs searches for these names.
- Harness comparison uses numeric tolerance: `compare_filtered()` in output.rs uses 1e-4 relative / 1e-15 absolute tolerance via `lines_match_approx()`. Falls back to interpolation-aware comparison when line counts differ (handles different adaptive timestep sequences between thevenin and ngspice).
- `.if/.elseif/.else/.endif` preprocessor: `process_conditionals()` in parse.rs, runs after `preprocess()`, before main parsing
- `.temp` directive: parsed to `Item::Temp(f64)`, retrieved via `thevenin::netlist_temp(&Netlist)` (default 27.0°C)
- Model-referenced resistors: `resolve_resistor_value()` in mna.rs checks Expr::Ident against model BTreeMap
- Subcircuit model scoping: `remap_model_ref()` in subckt.rs prefixes model names/refs with subcircuit instance path
- AC analysis: `simulate_ac(&Netlist) -> Result<SimResult>` in thevenin::ac
- AC uses `ComplexLinearSystem` (separate real/imag sparse matrices) solved via faer `c64` LU decomposition
- AC sweep modes: DEC (n pts/decade, log spacing), OCT (n pts/octave), LIN (n total points, linear)
- AC source excitation: `AcSpec { mag, phase }` on voltage/current sources → complex RHS in branch/node equations
- AC capacitor stamp: imaginary part += ωC (susceptance) between cap nodes
- AC inductor stamp: imaginary part += -ωL on branch diagonal (branch equation)
- AC diode stamp: small-signal conductance g_d at DC OP + junction capacitance Cj(V) + transit time cap τ*g_d
- `DiodeModel::junction_capacitance(v)` computes depletion cap Cj(V) = CJO/(1-V/VJ)^M with linear extrapolation beyond FC*VJ
- Adaptive timestep: LTE estimated from BE vs Trap difference on capacitor charges and inductor fluxes
- `BreakpointTable` collects discontinuity times from waveforms, forces timestep to land on them
- `waveform::breakpoints(&Waveform, &TranParams) -> Vec<f64>` extracts breakpoint times from waveforms
- Breakpoint behavior: at breakpoints, use BE integration and reduce step by 0.1× for stability
- `h_max` = tmax from .tran if specified, otherwise min(tstep, tstop/50) — caps internal timestep
- LTE tolerance: `TRTOL * max(vol_tol, chg_tol)` where TRTOL=7.0 (matching ngspice CKTtrtol)
- Timestep adjustment: reject if new_h < 0.9×step_h, accept and grow up to 2× otherwise
- step_h vs h: step_h is the actual step size used this iteration, h is the suggested next step from LTE
- Waveform evaluation: `waveform::evaluate(&Waveform, t, &TranParams) -> f64` in thevenin::waveform
- `TranParams { tstep, tstop }` provides defaults for waveform parameters (rise/fall times, SIN freq, etc.)
- `VoltageSourceInstance` and `CurrentSourceInstance` in MnaSystem track branch_idx/node indices + waveform
- In transient, after copying base stamps, waveform values override source RHS entries at each timestep
- PULSE defaults: td=0, tr=tstep, tf=tstep, pw=0, per=tr+pw+tf; repeats via time modulo period
- SIN phase parameter is in degrees (converted to radians internally), damping via exp(-theta*(t-td))
- Transient: `simulate_tran(&Netlist) -> Result<SimResult>` in thevenin::transient
- Transient uses BE for first timestep, then switches to Trapezoidal (matching ngspice)
- Capacitor companion model: BE → geq=C/h, ieq=-C/h*v_prev; Trap → geq=2C/h, ieq=-(2C/h*v_prev + i_prev)
- Inductor companion model: BE → req=L/h, veq=-L/h*i_prev; Trap → req=2L/h, veq=-(2L/h*i_prev + v_prev)
- Inductor in transient: adds -req on branch diagonal, veq on branch RHS (on top of topology stamps)
- `CapacitorInstance` and `InductorInstance` in MnaSystem hold resolved values and IC params
- IC= param on C/L elements overrides DC OP initial voltage/current for transient
- Transient output: "time" vector + node voltages + branch currents, plot name "tran1"
- DC sweep: `simulate_dc(&Netlist) -> Result<SimResult>` in thevenin::simulate
- DC sweep reuses assembled MNA matrix, only modifies RHS entries per sweep point
- Voltage source sweep: set `system.rhs[num_nodes + vsource_idx] = sweep_value`
- Current source sweep: undo old stamp, apply new value at node RHS entries
- DC sweep result has sweep variable as first vector (e.g., "v1"), then node voltages and branch currents
- Double sweep: outer loop is src2, inner loop is src1

## 2026-03-11 - BSIM3SOI partial fixes (iteration 27)
- Changes made:
  - Fixed NCH/NPEAK alias in all 3 BSIM3SOI models (DD/FD/PD): in ngspice 'nch' is an IOP alias for 'npeak' (same field). Our code had them as separate fields; model files set NCH=3.3e17 but npeak stayed at default 1.7e17. Fix: sync nch→npeak (or npeak→nch) after parameter parsing if one is at default and the other isn't.
  - Fixed vbi formula in all 3 models: used `npeak * 1e10 / ni²` but correct is `1e20 * npeak / ni²` (1e20 = n+ S/D doping ND). Reference: b3soiddtemp.c `log(1.0e20 * npeak / (ni*ni))`. Applied to both precompute() and size_dep_param() in DD and PD; precompute() only in FD.
  - Removed temporary debug prints (DD_DEBUG, FD_DEBUG env-var guards).
  - Fixed clippy::collapsible_if in all 3 models (floating body stability conductance block).
- Results: Still 15/15 bsim3soi harness tests failing; 10 other tests still passing (no regressions).
- Key findings from investigation:
  1. DD t3: 5.97µA vs 3.63µA (64% high). Root cause: vbseff=0.41V even when Vb=0 due to smooth-max(vbs_i, vbs0eff_dd+0.02) giving vbsdio≈vbs0eff_dd+0.02 when vbs_i < vbs0eff_dd+0.02. This is correct DD architecture but there may be another formula error in the vbs0eff chain.
  2. FD t3: 3.67µA vs 3.40µA (8% high). Better after our fixes. Root cause: FD should use vbsdio=vbs0eff_fd (NOT smooth-max with actual body voltage), but implementing this correctly requires NOT creating an internal body node for floating body devices. Attempted fix was reverted because it introduced a new failure (1.85e-4A at Vd=0) — the MNA matrix became near-singular with only 3e-12 S total conductance at the body node.
  3. PD: all 5 tests not yet deeply investigated.
  4. inv2/RampVg2: NR convergence failures not yet investigated.
- Recommendation: Split US-059 into 4 sub-stories (US-059a through US-059d). See updated notes in prd.json for US-059 for full details. This task is too large for a single Ralph pass — each sub-story is ~2-3 hours of work.
